# Base Branch
Do NOT use the base branch of main, unless explicitly stated by the developer. Base branch, by default, shall be staging. 

# PR Title Convention

Use one of these prefixes in the PR title:

- `feat:` — new feature
- `fix:` — bug fix
- `chore:` — maintenance task
- `docs:` — documentation-only change

# Rules

- Never merge the PR without human approval.
- Do not add Generated with Claude Code within the body of PR
- Do not add a Test Plan section
